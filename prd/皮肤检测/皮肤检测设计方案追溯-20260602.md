# 皮肤检测需求设计方案追溯

更新时间：2026-06-02  
适用范围：小程序皮肤检测、后端接口、数据库、管理后台、PDF 导出、部署配置

> 本文用于记录本轮皮肤检测需求从产品目标到技术实现的完整方案，便于后续开发、联调、部署和回溯。本文不记录真实密钥，仅记录配置项名称和部署方式。

---

## 1. 需求背景

用户希望在现有减脂小程序中新增“皮肤检测”能力，用户通过小程序上传照片并填写基础资料，系统调用第三方宜远智能测肤接口完成检测，再结合 DeepSeek 生成更适合营销转化的综合肤况分析和分项说明。

管理后台需要支持：

- 皮肤检测白名单和次数管理；
- 检测记录查询；
- 检测详情查看；
- 皮肤检测项配置；
- AI 提示词配置；
- 一键导出 PDF 报告。

小程序需要支持：

- 皮肤检测入口；
- 用户资料填写；
- 拍照/上传图片；
- 检测中 loading 动画；
- 综合报告页；
- 单项报告详情页；
- 定制方案客服弹层和二维码保存。

---

## 2. 产品目标

### 2.1 用户侧目标

1. 用户可以在小程序完成皮肤检测流程。
2. 用户可以看到综合肤况分和 6 项检测结果。
3. 用户可以点击每个检测项查看详细说明。
4. 用户可以通过“定制方案”添加客服微信。
5. 客服二维码可保存到相册，便于转化。

### 2.2 管理后台目标

1. 管理员可以维护皮肤检测白名单。
2. 管理员可以调整每个手机号的皮肤检测次数。
3. 管理员可以查询所有检测记录。
4. 管理员可以查看单条记录详情。
5. 管理员可以配置检测项启用状态、展示名、分档规则和提示词 key。
6. 管理员可以配置综合分析和分项分析提示词。
7. 管理员可以导出结构化 PDF 报告。

### 2.3 营销目标

1. 检测报告用于引导用户关注自身肤况问题。
2. 单项分析需要有“诊断结论、产生原因、护理建议”。
3. 报告和客服弹层需要引导用户添加客服微信。
4. PDF 首页需要展示品牌 logo、客服二维码和客服微信号。

## 4. 数据库设计方案

本次皮肤检测相关数据库设计分为三部分：基础记录、用户次数、配置化能力。

### 4.1 基础记录与白名单

迁移文件：

```text
database/migrations/V026__skin_detection.sql
```

主要职责：

- 皮肤检测白名单；
- 检测次数；
- 检测记录；
- 检测明细。

核心表设计方向：

```text
skin_detection_whitelist
skin_detection_quota_log
skin_detection_record
skin_detection_record_item
```

设计原则：

1. 白名单与减脂/拍照识别白名单独立，不复用原有次数体系。
2. 每个手机号有独立的总次数和已使用次数。
3. 每次检测生成一条检测记录。
4. 6 项检测结果以明细形式保存，便于后台查询、PDF 导出和后续扩展。

### 4.2 用户资料补充

迁移文件：

```text
database/migrations/V027__user_profile_birthday_region.sql
```

主要职责：

- 补充用户生日；
- 补充用户居住地/地区信息；
- 支持皮肤检测所需年龄、性别、地区字段。

### 4.3 检测项配置与 AI 配置

迁移文件：

```text
database/migrations/V028__skin_detection_config_ai.sql
```

主要职责：

1. 检测项配置化；
2. DeepSeek 提示词配置化；
3. 检测明细增加 AI 结构化字段。

核心能力：

```text
skin_detection_item_config
skin_detection_ai_prompt_template
skin_detection_record_item.ai_conclusion
skin_detection_record_item.ai_reason
skin_detection_record_item.ai_care_advice
skin_detection_record_item.ai_status
skin_detection_record_item.ai_error
skin_detection_record_item.ai_raw_json
```

设计原则：

1. 6 项检测项不写死在前端。
2. 后端可通过配置决定检测项顺序、展示名、启用状态和提示词 key。
3. AI 输出按结构化字段保存，避免只保存一段不可解析文本。
4. DeepSeek 调用失败不影响宜远检测结果保存。
5. AI 字段为空时，前端和 PDF 均有兜底文案。

## 5. 后端设计方案

### 5.1 配置项设计

配置前缀：

```yaml
app:
  skin-detection:
```

环境变量映射：

```text
YIMEI_API_BASE_URL
YIMEI_CLIENT_ID
YIMEI_CLIENT_SECRET
YIMEI_FACE_DETECT_TYPES
DEEPSEEK_BASE_URL
DEEPSEEK_API_KEY
SKIN_DETECT_DEFAULT_TIMES
SKIN_DETECT_IMAGE_DIR_SUFFIX
```

对应配置：

```yaml
app:
  skin-detection:
    yimei-base-url: ${YIMEI_API_BASE_URL:https://api.yimei.ai}
    yimei-client-id: ${YIMEI_CLIENT_ID:change-me}
    yimei-client-secret: ${YIMEI_CLIENT_SECRET:change-me}
    detect-types: ${YIMEI_FACE_DETECT_TYPES:106510}
    deepseek-base-url: ${DEEPSEEK_BASE_URL:https://api.deepseek.com}
    deepseek-api-key: ${DEEPSEEK_API_KEY:change-me}
    default-times: ${SKIN_DETECT_DEFAULT_TIMES:3}
    image-dir-suffix: ${SKIN_DETECT_IMAGE_DIR_SUFFIX:skin-detection}
```

安全要求：

- 真实密钥只放服务器环境变量或 `application-local.yml`；
- 不提交真实密钥到仓库；
- 文档中只记录变量名，不记录真实值。

### 5.2 后端核心服务

主要文件：

```text
backend/src/main/java/com/ypfr/loseweight/service/SkinDetectionService.java
backend/src/main/java/com/ypfr/loseweight/service/DeepSeekSkinAnalysisClient.java
backend/src/main/java/com/ypfr/loseweight/service/SkinDetectionPdfService.java
backend/src/main/java/com/ypfr/loseweight/config/SkinDetectionProperties.java
```

职责划分：

#### SkinDetectionService

负责：

- 检测次数校验；
- 图片保存；
- 宜远接口调用；
- 记录和明细落库；
- DeepSeek 分析调度；
- 小程序报告数据组装；
- 后台详情数据组装。

#### DeepSeekSkinAnalysisClient

负责：

- 读取综合提示词和分项提示词；
- 构建请求；
- 调用 DeepSeek；
- 解析结构化 JSON；
- 返回综合结论和分项字段。

#### SkinDetectionPdfService

负责：

- 生成 PDF；
- 展示品牌 logo；
- 展示客服二维码和微信号；
- 输出结构化报告内容；
- 隐藏技术字段和原始 JSON。

### 5.3 第三方接口调用策略

#### 宜远智能测肤

用途：

- 获取基础皮肤检测结果；
- 获取每个检测项分数、等级、图片和原始指标。

设计：

- 接口凭据由环境变量提供；
- 检测项默认通过 `YIMEI_FACE_DETECT_TYPES` 提供兜底；
- 后续可通过数据库启用项动态计算检测组合。

#### DeepSeek

用途：

- 生成综合结论；
- 生成每个检测项的结构化分析。

输出字段：

```text
aiSummary
aiConclusion
aiReason
aiCareAdvice
aiStatus
aiError
aiRawJson
```

失败策略：

- DeepSeek 未配置时，状态为 skipped；
- DeepSeek 调用失败时，状态为 failed；
- 不影响主检测记录生成；
- 前端/PDF 使用宜远结果或兜底文案展示。

## 6. 小程序设计方案

### 6.1 页面范围

主要页面：

```text
frontend/src/pages/skin-detection/report.vue
frontend/src/pages/skin-detection/detail.vue
frontend/src/pages/skin-detection/loading.vue
```

### 6.2 综合报告页

职责：

- 展示综合肤况分；
- 展示综合结论；
- 展示 6 项检测结果入口；
- 展示“定制方案”按钮；
- 弹出客服二维码和微信号；
- 支持保存二维码到相册。

客服信息来源：

```text
GET /api/v1/skin-detection/customer-service/current
```

该接口复用现有客服配置：

```text
serviceWechat
serviceQrImageUrl
serviceQrImagePath
servicePhone
```

展示规则：

1. 有二维码时显示二维码和保存按钮。
2. 没有二维码时显示“暂无客服二维码”。
3. 二维码加载失败时也显示占位，不影响微信号展示。

### 6.3 单项报告详情页

职责：

- 展示检测项名称；
- 展示单项分数；
- 展示等级；
- 展示结构化分析。

展示字段：

```text
诊断结论：aiConclusion / resultText / 兜底文案
产生原因：aiReason / 兜底文案
护理建议：aiCareAdvice / 兜底文案
```

已明确隐藏：

```text
检测项编码
分档类型
接口等级
metricsJson 原始数据
```

设计原则：

- 小程序用户端不展示技术字段；
- 不展示原始 JSON；
- 分数圆圈需居中；
- 文案使用业务化语言，不直接暴露 AI/接口状态。

### 6.4 Loading 页面

职责：

- 检测中状态展示；
- 保持用户等待感；
- 加强动画效果；
- 不改变检测流程和跳转逻辑。

## 7. 管理后台设计方案

### 7.1 页面范围

主要页面：

```text
admin-frontend/src/views/SkinDetectionRecordView.vue
admin-frontend/src/views/SkinDetectionWhitelistView.vue
admin-frontend/src/views/SkinDetectionItemConfigView.vue
admin-frontend/src/views/SkinDetectionAiPromptTemplateView.vue
```

### 7.2 皮肤检测记录页

功能：

- 按手机号、状态、时间、综合分筛选；
- 查看记录列表；
- 查看详情；
- 导出 PDF。

详情弹窗展示：

```text
基础用户信息
检测图片
综合分析
6 项检测明细
```

图片处理：

- 支持完整 URL；
- 支持相对上传路径；
- 图片失败时显示“图片加载失败”，不显示浏览器默认 FAILED。

### 7.3 检测明细说明列

需求：

后台详情表格“说明”列不能只显示单个字段，需要展示：

```text
诊断结论 + 产生原因 + 护理建议
```

展示格式：

```text
诊断结论：xxx
产生原因：xxx
护理建议：xxx
```

规则：

1. 有哪个字段显示哪个字段；
2. 字段为空时跳过该段；
3. 多段之间换行；
4. 结构化字段都没有时回退到 `resultText` 或 `aiError`；
5. 表格单元格使用 `white-space: pre-line` 支持换行。

### 7.4 PDF 文件名

原始文件名：

```text
skin-detection-report-{id}.pdf
```

调整后：

```text
yyyyMMdd-用户姓名-皮肤诊断报告.pdf
```

示例：

```text
20260602-瞿炜-皮肤诊断报告.pdf
```

姓名为空时兜底：

```text
20260602-用户6-皮肤诊断报告.pdf
```

兼容性注意：

- 管理后台构建目标不支持 `String.replaceAll`；
- 文件名日期格式化使用 `replace(/-/g, '')`。

## 8. PDF 导出设计方案

### 8.1 PDF 首页布局

顶部采用三栏布局：

```text
左侧：宝护健康瘦 logo
中间：皮肤检测结构化报告 + 报告编号
右侧：客服二维码 + 客服微信号
```

### 8.2 Logo 资源方案

正确源文件：

```text
frontend/src/static/logo/logo-c-dark.png
```

后端 PDF 使用文件：

```text
backend/src/main/resources/static/pdf/baohu-logo.png
```

导出尺寸：

```text
256 × 256
```

读取方式：

```text
classpath:/static/pdf/baohu-logo.png
```

部署说明：

- logo 放入后端 resources；
- Maven 打包时进入 jar；
- 服务器无需单独上传 logo；
- 重新打包 jar 后即可生效。

### 8.3 客服二维码与微信号方案

客服配置复用现有客服配置服务：

```text
PhotoRecognitionAccessService.getServiceConfig()
```

读取字段：

```text
serviceWechat
serviceQrImagePath
serviceQrImageUrl
status
```

二维码读取优先级：

```text
serviceQrImagePath 本地上传路径
-> serviceQrImageUrl 本地上传相对路径
-> serviceQrImageUrl http/https URL
```

失败策略：

- 二维码读取失败不影响 PDF 导出；
- 微信号有值时仍然显示；
- 客服配置禁用时不展示二维码。

### 8.4 PDF 内容结构

当前 PDF 结构：

```text
顶部品牌区
一、报告概览
二、综合结论
三、重点关注项
四、分项结构化分析
五、免责声明
```

### 8.5 PDF 内容展示规则

已调整：

1. `AI 结论` 改为 `诊断结论`；
2. 隐藏 `AI 状态`；
3. 隐藏 `检测数据摘要`；
4. 删除免责声明中的“第三方接口密钥”相关句子；
5. 不展示原始 JSON；
6. 不展示第三方接口技术字段。

免责声明：

```text
本报告由 AI 皮肤检测能力生成，仅用于日常护肤参考，不作为医疗诊断依据。如存在严重皮肤问题，请咨询专业医生。
```

## 12. 风险与注意事项

### 12.1 第三方接口风险

- 宜远接口字段可能变化；
- DeepSeek 输出可能不是合法 JSON；
- 第三方接口超时会影响用户体验。

应对：

- 后端保留兜底字段；
- DeepSeek 失败不影响主检测结果；
- 提示词配置可在后台调整。

### 12.2 图片路径风险

- 本地开发和服务器部署目录不同；
- PDF 读取二维码时需要把相对上传路径解析为服务器本地路径；
- 图片不存在时不能导致 PDF 导出失败。

应对：

- 上传根目录使用 `app.upload.avatar-dir`；
- PDF 二维码读取失败时跳过图片；
- 微信号仍然显示。

### 12.3 前端兼容风险

- 管理后台当前 TS lib 不支持 `String.replaceAll`；
- 后续代码中应避免使用新 API，除非同步调整 tsconfig lib。

应对：

- 日期字符串替换使用 `replace(/-/g, '')`。

### 12.4 敏感信息风险

- 需求简要原始文档中出现过真实密钥；
- 后续提交前应避免继续扩散真实密钥。

应对：

- 新文档不记录真实密钥；
- 真实密钥只放服务器环境变量或本地忽略配置。
