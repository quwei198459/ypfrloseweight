# 一、TDEE = BMR × 活动系数 是什么意思

## 1️⃣ BMR（基础代谢）

👉 **人在“什么都不做”时每天消耗的能量**

比如你躺着一天：

* 呼吸
* 心跳
* 维持体温
* 器官运转

这些都会消耗能量

👉 这部分就是 **BMR（基础代谢率）**

---

## 2️⃣ TDEE（总消耗）

👉 **人在真实生活中一天总共消耗的能量**

因为现实中你不会躺着：

* 走路
* 上班
* 打字
* 做饭
* 运动

这些都会增加消耗

👉 所以：

👉 **TDEE = BMR + 日常活动消耗 + 运动消耗**

---

# 二、为什么用“乘以活动系数”

现实问题：

👉 我们很难精确计算“你每天到底走了多少路、干了多少活”

所以行业通用做法是：

👉 用一个“活动系数”来估算

---

## 常见活动系数（你系统里也是这个逻辑）

| 活动等级 | 含义       | 系数    |
| ---- | -------- | ----- |
| 久坐   | 几乎不动     | 1.2   |
| 轻度活动 | 偶尔走动     | 1.375 |
| 中度活动 | 每周运动3-5次 | 1.55  |
| 高强度  | 经常运动     | 1.725 |
| 非常高  | 体力劳动/运动员 | 1.9   |

👉 所以：

👉 **TDEE = BMR × 活动系数**

本质就是：

👉 用一个倍率，把“静态消耗”扩展成“真实生活消耗”

---

# 三、举个例子（你系统可以直接用）

假设一个用户：

* BMR = 1500 kcal
* 活动系数 = 1.375（轻度活动）

那么：

👉 TDEE = 1500 × 1.375 = 2062 kcal

---

👉 含义：

👉 **这个人每天大概会消耗 2062 kcal**

# 四、它在你系统里的作用（非常关键）

这个值直接决定：

## 1️⃣ 减脂逻辑核心

👉 如果：

摄入 < TDEE → 减脂
摄入 > TDEE → 增重

---

## 2️⃣ 预算计算（你系统核心）

你现在的公式是：

👉 daily_budget = TDEE - 热量缺口

比如：

* TDEE = 2000
* 缺口 = 500

👉 daily_budget = 1500

---

👉 含义：

👉 **你每天只能吃 1500 kcal 才能瘦**

---

## 3️⃣ 实际缺口计算

你系统里还有：

👉 actual_deficit = TDEE + 运动 - 摄入

---

👉 这里 TDEE 是“基础真实消耗”

### 2️⃣ 推荐是否可信

### 3️⃣ 数据是否一致

你所有这些都依赖它：

* 热量缺口

* 趋势分析

# 六、你系统里一个“容易踩坑点”（重点）

👉 **活动系数是静态的**

但现实是：

* 有些人今天运动很多
* 有些人今天几乎没动

---

## 你现在模型：

TDEE = 固定值（每天一样）

运动单独加：

👉 actual_deficit = TDEE + exercise - intake

---

## 更高级模型（后续可以升级）

👉 动态 TDEE：

TDEE = BMR × 活动系数（基础） + 实时活动
