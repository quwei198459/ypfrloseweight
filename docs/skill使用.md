模板 1：修 bug

启用 karpathy-guidelines + bugfix-minimal-change。

目标：
修复当前问题，但不要扩大改动范围。

要求：
1. 先复述现象、预期结果、复现方式
2. 缩小原因范围，给出最可能根因
3. 只修改与问题直接相关的文件
4. 不要顺手改样式、结构、接口协议
5. 输出修改点、验证步骤、回归检查点

模板 2：按设计稿修页面
启用 karpathy-guidelines + design-to-code-guard。

目标：
根据设计稿/截图修复页面，高保真还原，不改变无关结构和交互。

要求：
1. 先拆页面区块与差异点
2. 说明本次只修哪些点
3. 保持现有页面结构和交互路径
4. 不要擅自改版
5. 完成后给设计稿对比验证清单

模板 3：后端接口改动
启用 karpathy-guidelines，并遵守 backend-safety-rules。

目标：
完成本次后端接口/数据处理改动，但确保兼容现有前端。

要求：
1. 先说明当前逻辑与目标逻辑
2. 列出涉及接口、字段、文件
3. 给出最小改动方案
4. 不要无关重构
5. 输出验证方式、示例请求、数据库检查方式
